Pressured Progression is a soccer analytics case study on how teams manage pressure in two moments: building from deep and attacking immediately after regains.

I built it as an end-to-end technical project using Python, XGBoost, SHAP, StatsBomb Open Data, ASA API data, and a Streamlit multi-page app. The thesis is simple: profile pressure-related patterns with explicit uncertainty and associational language, then compare across teams without making causal coaching claims.

Deliverables include: reproducible data pipelines, labeled sequence marts, interpretable ML diagnostics, bootstrap confidence-interval visualizations, an interactive Streamlit interface, and a long-form written analysis tied to chart-level evidence.

The project is intentionally honest about limits (small sample windows, cross-league context differences, observational data constraints) while still producing clear, reviewable outputs for technical and non-technical readers. It demonstrates my approach to scoped research communication: quantify what can be measured, state what cannot, and keep assumptions visible.
